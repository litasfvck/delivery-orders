# Грузопоток — фронтенд приёмки заказов на доставку

Реализация клиентской части ТЗ «Приёмка заказа на доставку»: React 18 + Vite,
маршрутизация через `react-router-dom`, запросы через `axios`, стили —
SCSS-модули (без UI-библиотек, вся вёрстка своя).

## Экраны

1. **`/orders/new`** — форма создания заказа. Все 6 полей обязательны
   (город/адрес отправителя, город/адрес получателя, вес груза, дата забора).
   Валидация выполняется на клиенте перед отправкой; дата забора не может
   быть в прошлом.
2. **`/orders`** — список всех заказов. Каждая карточка показывает
   автоматически присвоенный номер заказа, маршрут «город → город», адреса
   и вес. Клик по карточке открывает просмотр.
3. **`/orders/:orderId`** — карточка заказа в режиме только для чтения.

## Запуск

```bash
npm install
cp .env.example .env.local   # указать адрес бэкенда
npm run dev
```

Приложение поднимется на `http://localhost:5173`.

### Локальная демонстрация без бэкенда

Пока ASP.NET API не готов, можно посмотреть вёрстку на встроенном
in-memory моке:

```
# .env.local
VITE_USE_MOCK_API=true
```

Мок лежит в `src/api/mockOrdersStore.js`, содержит несколько тестовых
заказов и имитирует сетевые задержки. Как только бэкенд будет готов —
просто уберите/поставьте `false` этому флагу.

## Контракт REST API (для интеграции с бэкендом)

Ожидаемый бэкендом контракт описан и задокументирован в
`src/api/ordersApi.js`:

```
GET    /api/orders          -> Order[]
GET    /api/orders/{id}     -> Order
POST   /api/orders          <- CreateOrderPayload   -> Order
```

```ts
type Order = {
  id: number | string;        // автогенерируемый номер заказа
  senderCity: string;
  senderAddress: string;
  recipientCity: string;
  recipientAddress: string;
  weightKg: number;
  pickupDate: string;         // yyyy-MM-dd
};

type CreateOrderPayload = Omit<Order, 'id'>;
```

Базовый адрес API берётся из `VITE_API_BASE_URL` (см. `.env.example`).
Если бэкенд отдаёт другие имена полей — проще всего поправить их прямо
в `src/api/ordersApi.js`, остальной код на это не завязан.

## Структура проекта

```
src/
  api/            axios-клиент и слой доступа к заказам (+ мок)
  components/     переиспользуемые UI-компоненты (кнопка, поле, бейдж номера,
                   индикатор маршрута, состояния загрузки/ошибки, layout)
  pages/          три экрана: CreateOrderPage, OrdersListPage, OrderDetailsPage
  hooks/          useAsync — загрузка данных с состояниями loading/error
  utils/          валидация формы, форматирование даты/веса
  styles/         глобальные токены дизайна (SCSS custom properties)
```

Каждый компонент/страница стилизованы через одноимённый `*.module.scss`
(CSS Modules), общие цвета/шрифты/радиусы — через CSS-переменные в
`src/styles/_tokens.scss`, чтобы визуальный язык был единым.
